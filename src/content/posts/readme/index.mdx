The following files are compatible with this document:

- `src/content/site.json` - homepage and site content source of truth.
- `src/lib/site-schema.ts` - validates the `site.json` structure.
- `src/lib/site-data.ts` - loads the `default` site entry.
- `src/pages/index.astro` - homepage template.
- `src/components/base/HeroMetric.astro` - hero metric display component.
- `src/components/base/SocialLink.astro` - social icon link component.
- `src/config.ts` - theme behavior and posts config.

### Site Content

Homepage content now lives in `src/content/site.json` using the same `head/body/sections` structure generated by the Spotren app.

```json title="site.json"
{
  "default": {
    "head": {
      "title": "Litos",
      "description": "Your site description",
      "website": "https://example.com",
      "lang": "en",
      "ogImage": { "url": "/og-image.webp" }
    },
    "body": {
      "brand": "Northwind",
      "category": "Dental Studio",
      "city": "Portland",
      "tagline": "Short homepage description",
      "rating": 4.9,
      "reviewCount": 50
    }
  }
}
```

The `head` object controls shared metadata used in layout, head tags, and feeds. The `body` object controls homepage content.

### Navigation Links

Header and footer navigation now come from `src/config.ts`:

```ts title="config.ts"
export const NAVIGATION_CONFIG = {
  headerLinks: [{ name: 'Posts', url: '/posts' }],
  footerLinks: [
    { name: 'Readme', url: '/' },
    { name: 'Posts', url: '/posts' },
    { name: 'Tags', url: '/tags' },
  ]
}
```

### Hero Content

The homepage hero reads from `body`, while fixed labels and iconography come from `src/config.ts`.

```json title="site.json"
{
  "body": {
    "brand": "Northwind",
    "category": "Dental Studio",
    "tagline": "Short homepage description"
    "rating": 4.9,
    "reviewCount": 50,
    "openStatus": "Aberto agora",
    "address": "1284 Market Street, Suite 300, Portland, OR 97205",
    "phone": "(503) 555-0147"
  }
}
```

The hero description is truncated to 150 characters by `truncateText()` before rendering.

### Social Links

Social icons below the hero come from `src/content/site.json`. Icons are from :link[Iconify]{id=https://icon-sets.iconify.design/}.

```json title="site.json"
{
  "socialLinks": [
    {
      "name": "facebook",
      "url": "https://facebook.com/yourpage"
    }
  ]
}
```

The icon is resolved by network key in `src/config.ts`, so the JSON only needs the social network `name` and `url`.

### Home Sections

Additional homepage sections are configured in `body.sections` and rendered above the posts list.

```json title="site.json"
{
  "body": {
    "sections": [
      {
        "key": "location",
        "title": "Location",
        "description": "",
        "latitude": 45.5231,
        "longitude": -122.6765,
        "googleMapsUri": "https://www.google.com/maps/place/?q=place_id:123"
      }
    ]
  }
}
```

Supported section keys are `location`, `reviews`, `products`, `services`, and `faqs`.

### Posts

The homepage still renders pinned posts first. If there are no pinned posts, it renders the latest posts using `POSTS_CONFIG.homePageConfig`.
